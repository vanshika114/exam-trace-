How it works

Three layers of verification

Not just logs — proof. Every submission is forensically traceable from first keystroke to final verdict.

01 / CLIENT
🔑

Keystroke Merkle tree

Every keydown and keyup event is timestamped and SHA-256 hashed. Leaves combine into a Merkle root — a single fingerprint of the entire typing session. Computed in the browser using Web Crypto API.

02 / SERVER
🔒

Server-side root validation

Express recomputes the Merkle root independently from the submitted keystroke log. If the roots don't match — even one altered timestamp — the session is flagged as tampered before analysis begins.

03 / ANALYZER
🧠

Behavioral + AST analysis

Python FastAPI scores hold times, inter-key gaps, and burst patterns. A separate AST difflib pass compares submitted code against known solution structures to detect paste-from-outside events.

Features

Everything the judge needs

Built for exam administrators who need to make defensible decisions, not guesses.

🌳

Merkle integrity verification

Client and server Merkle roots are displayed side-by-side on the judge dashboard. A mismatch is automatic grounds for FLAGGED status — no human judgment needed.

📊

Live keystroke chart

Inter-key gap chart renders in real time during the exam. Red spikes mark sub-30ms bursts — statistically impossible for human typing, consistent with injected keystrokes.

🔍

AST similarity scoring

Submitted code is parsed into an AST node histogram and compared against known solutions via cosine similarity. 60%+ match triggers SUSPICIOUS; 85%+ triggers FLAGGED.

👁

Tab exit detection

Every time the candidate switches away from the exam tab, it's logged with a timestamp. The count is displayed on the editor in real time and stored in the session record.

📋

Paste event tracking

Paste events are intercepted and counted separately from normal typing. Combined with the keystroke burst analyzer, this catches copy-paste even when candidates paste character by character.

⚖️

Three-tier verdicts

CLEAN, SUSPICIOUS, and FLAGGED — each with the exact metrics that triggered the verdict. Judges see anomaly score, hold time mean/std, IKG distribution, and AST similarity in one view.

Verdicts

Clear decisions. Documented reasons.

CLEAN

Normal hold times, consistent IKG, low anomaly score, Merkle valid

SUSPICIOUS

Anomaly score >0.45 or AST similarity >60% — warrants review

FLAGGED

Merkle mismatch, anomaly >0.75, or AST similarity >85%
